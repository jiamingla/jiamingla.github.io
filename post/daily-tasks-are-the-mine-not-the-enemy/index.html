日常任務不是主線的對手，是主線的礦場——一位前輩的「每天擠一支 script」，與 AI 時代的新版本

我最近常有一種焦慮：一整天被例行的手動測試填滿，感覺有在產出，但那些「真正重要、想好好思考的事」，卻一直沒空碰。

說來有點不好意思——我才剛寫完一篇講 POM 重構的技術文章，談「把測試寫成能被別人接手的形狀」，結果回到日常，還是被「手動測試把時間都吃光了」這件事困住。

如果你也有這種感覺，這篇是寫給你的。我想講一個前輩教我的小方法，以及它在 AI 時代為什麼不但沒過時，還升級了。

一、一位前輩教我的事：每天，還是擠得出一支 script

幾年前在前一份工作，一位資深的 QA 前輩跟我說過一句話，我到現在還記得：

「手動測試真的很花時間，這我知道。但你每天還是擠得出一點時間，寫一兩支 script。」

當時我只當它是一句勵志的話。後來才慢慢懂——這句話真正厲害的地方，不是「叫你更勤勞」，而是它悄悄否定了一個我們都信的假設：

「等我有一整塊空下來的時間，再來好好做重要的事。」

這個假設的問題在於：那一整塊時間永遠不會來。 旺季有交付、淡季有別的事，你等的那個「終於有空」的早上，大概到離職都等不到。

我後來才意識到，這個假設背後其實藏著一種完美主義：總覺得要「到了某個位置」、「等到有一整塊乾淨的時間」，才有資格好好做那件重要的事。

但寫腳本這件事，其實比較像彈吉他，或像打掃廁所——每天碰一點，遠比一個禮拜卯起來做一整天有效。 吉他三天不碰手就生了，廁所放著不管就髒到你不想動手；腳本的累積也一樣，靠的是頻率，不是哪一次的爆發量。你不會說「等我有一整個週末，再來好好彈吉他」——你就是每天撥個幾下。重要的事，也該這樣對待。

前輩的話之所以有用，是因為它換了一個前提：不要等整塊，每天擠一條縫就好。 一天一支 script，一年就是兩百多支；而你要是一直等「整塊時間」，一年下來是零。

二、為什麼這招有效：日常不是你的對手，是你的礦場

很多人把「日常任務」當成「重要的事」的對手——好像日常是來搶你時間的敵人，你得先把它打發掉，才能去做真正有價值的事。

但我後來發現，事情剛好相反：

日常任務不是主線的對手，是主線的礦場。

你想推進的那些「重要的事」，大部分其實就埋在日常任務裡。你本來就要跟 PM 對需求、本來就要碰某個資料欄位、本來就要問後端某個防禦機制——這些日常動作底下，全藏著你想思考的主線問題。你需要的不是「另外挪出時間」，是做這些日常時，記得用主線的眼光看它一眼。

所以真正稀缺的，從來不是「一整塊空下來的時間」，而是那幾秒鐘的意識——意識到「我手上這件例行公事，其實連著一條我在乎的線」。

這也順便拆掉一個焦慮。我們常焦慮的是「會不會接下來三個月都這麼忙、都沒空成長」，但如果你誠實問自己「今天這些事，做得完嗎？」答案通常是做得完的。你焦慮的多半是『想像中的忙』，不是『今天真實的忙』。 而成長，本來就不需要等到不忙。

三、AI 時代，這招升級了：從「自己寫」到「盯著 agent 寫」

那位前輩的話是好幾年前說的，那時候「擠時間寫 script」指的是你自己打開編輯器，一行一行敲。

到了 AI 時代，我覺得這句話不但沒過時，還升級了——你每天擠的那一點時間還在，只是「自己寫一兩支 script」，變成「盯著 agent 幫你寫一兩支 script」。

你的角色從「動手敲的人」，變成「定義要什麼、並盯著它有沒有寫對的人」。這聽起來像是輕鬆了，其實責任更集中了：agent 可以很快生出十支 script，但「這些 script 到底有沒有測到對的東西」「它寫的斷言對不對」——這還是人的事，而且更是人的事。

這跟我在 POM 那篇講的人機分工是同一件事：AI 負責機械的部分（找定位、拼程式、產生大量案例），人負責判斷的部分（要測什麼、什麼才算對）。 工具換了、速度快了，但「每天擠一點、讓它複利」這個動作的本質沒變——變的只是你擠出來的那點時間，現在拿去「盯場」而不是「敲鍵盤」。

對一個被手動測試塞滿的 QA 來說，這其實是好消息：盯著 agent 寫，比自己從零寫，更擠得進日常的縫裡。

四、把它變成每天的習慣（旺季也能做的版本）

你不需要一個偉大的計畫，也不需要等淡季。下面是可以明天就開始、而且旺季也不會把你壓垮的做法：

  1. 做一件例行任務前，停三秒問一句：「這個日常任務，在餵哪一條我在乎的主線？」看一眼就好，不必當場動手。這幾秒鐘，就是你在礦場裡看見礦脈的那一眼。
  2. 把挖到的記下來，當成「將來的積木候選」：不是現在就做，是先讓清單長出來。哪個 case 你每次都要手動重跑？記下來，那就是下一支 script（或下一個你要丟給 agent 的任務）的候選。
  3. 每天擠一條縫：自己寫一支也好、盯 agent 寫一支也好。重點不是當天的產量，是讓它每天發生。
  4. 但別因為「被擠壓的挫折」就衝動開大工程：累積的疲憊加上當下的煩躁，最容易讓人想「乾脆停下來搭一套大工具一勞永逸」。那通常是紅燈，不是時機。先記下念頭，回去做眼前的事。每天一條縫的複利，會比一次性的大工程更可靠。

結語：稀缺的不是時間，是那一眼

說到底，這篇講的東西，跟我一直在談的問題驅動測試是同一個習慣，只是把它從「測試對象」搬到了「你自己的工作流」——對每一件日常任務，問它一個問題：你連著哪一條重要的線？

那位前輩沒有教我什麼高深的技術，他只是讓我看見：重要的事不是在你「終於有空」的那天才開始的，是在你每天那條被擠出來的縫裡，一點一點長出來的。

而到了 AI 時代，這條縫變得更好擠了。你不必自己敲完每一行——你只要每天，盯著它，往前推一點點。
