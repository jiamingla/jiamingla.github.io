【開台宣言】測試的靈魂在於提問：我未來想分享的寫作藍圖

  2026-01-29   2026-01-29

大家晚安！

這段時間我的更新頻率會稍微放慢一些，主要在處理搬家的各項瑣事。不過，這段空檔也讓我能靜下心來梳理未來的寫作方向。

我一直在思考，除了單純的技術工具，測試工程師還能為團隊帶來什麼價值？未來，我希望透過這個部落格，紀錄我在「品質工程」這條路上的探索。以下是我初步規劃的系列文章藍圖：

一、 核心思維：從執行者轉向思考者

  • 問題驅動測試 (PDT) 專題： 究竟什麼是問題驅動測試？它與我們熟捻的「腳本化測試」有何本質上的差異？為什麼在複雜系統中，PDT 才是抓出致命 Bug 的關鍵？
  • 推動變革： 在資源有限的環境下，我們該如何有策略地推動單元測試與問題驅動測試？

二、 技術實踐：現代工具與架構優化

  • AI 驅動開發： 分享如何利用 Claude Code + MCP 的組合，高效撰寫 Android App 前端自動化測試（Python）腳本。
  • 架構設計： 深入探討 Page Object Pattern (POM)，分享如何將雜亂無章的測試腳本，優雅地重構成易於維護的形狀。

三、 職場生存與價值導向

  • 期待管理： 「領多少錢做多少事」是真的嗎？作為一名測試員，我該對工作抱持什麼樣的成長期待？
  • 利害關係人溝通： 如何在紛亂的需求中，精準搞清楚開發、產品、主管等利害關係人的真實期待？

四、 信仰與工作的對話

  • 基督徒的測試視角： 作為一名基督徒，在軟體測試這份強調「挑錯」與「嚴謹」的工作中，我可以期待什麼樣的召命與價值？

希望這些內容能為同樣在測試領域奮鬥的你，帶來一點啟發與共鳴。搬家事宜告一段落後，我會儘快開始第一篇的更新，敬請期待。

一切順利。
