# 〈日常任務不是主線的對手，是主線的礦場〉骨架

> 📂 **定位**：下一篇候選（2026-06-25 user 拍板選 B）。低能量時期理想下一篇——素材已備、無指紋、actionable、**非純線六**（線六職涯 × 線一 PDT meta）。
> 🎯 **主軸**：成長不在「擠出整塊時間想大事」，在「**做日常任務時用主線眼光看一眼**」。稀缺的不是綠燈時段，是那幾秒的**意識**。
> 🪙 **金句**：「日常任務不是主線的對手，是主線的礦場。」
> 📦 **素材來源**：[coach-sessions/2026-06-10-insights.md](coach-sessions/2026-06-10-insights.md)（信念A假 / 旺季淡季 / 礦場原句）+ [coach-sessions/2026-06-25-impostor-and-manual-squeeze.md](coach-sessions/2026-06-25-impostor-and-manual-squeeze.md)（今天的活素材）。

## ⚓ 誠實錨 / 開場鉤（候選）

> **連我自己——一個寫部落格談測試方法論的人、剛發完一篇 POM 重構文——都還在為「整天做手動測試、沒時間思考重要的事」而焦慮。**

這個鉤子的力量：不裝高人指路，而是「我也卡在這、以下是我挖出來的出路」。共鳴 > 說教。（接 memory `feedback_self_assessment_pattern`：用自己當例子最誠實。）

---

## 🧩 核心結構（草，4 段）

### 1. 那個熟悉的焦慮：日常把思考擠掉了
- 場景：一天被例行任務填滿，感覺有產出，但「真正重要的那些事」一直沒空想。
- 命名出「假信念那半」：**忙＝價值**。日常填滿時間→感覺有產出≠重要任務有推進。

### 2. 翻轉：日常不是對手，是礦場
- 大部分「想思考的主線問題」其實**搭車在日常任務上**——你本來就要做那件日常的事，順手用主線眼光看一眼，就挖到了。
- **稀缺的不是時間，是意識**：做日常任務時記得用主線眼光看一眼（幾秒）。
- ⚠️ 去識別化：原始例子（型一型二可信度 / test case 欄位命名 / 負向測試）要抽象成通用版——「你本來就要跟 PM 對需求／本來就要碰某個資料欄位／本來就要問後端防禦」。別寫公司專有名詞。

### 3. 這其實是 PDT 的日常版（接招牌主張）
- 用主線眼光看日常 = 對每個日常任務問一個問題：「這在餵哪條重要的線？」
- 這正是[問題驅動測試](/post/problem-driven-testing-intro/)的 meta 版——把「提問」的習慣從測試對象，搬到自己的工作流。

### 4. 給讀者的具體做法（actionable，旺季也能用）
1. 做一件例行任務前，停 3 秒問：「這個日常任務在餵哪條主線？」看一眼就好，不必當場做。
2. 把挖到的記下來當「將來積木候選」清單——不是現在做，是有空再做。先讓清單長出來。
3. 把「我被日常擠壓」改問「今天真實做得完嗎」（想像的忙 vs 真實的忙），焦慮會小。
4. ⚠️ 別因擠壓的挫折就衝動開大工具專案（紅燈）——下一步要小、要搭車在日常上。

## ⚠️ 注意

- **非純線六**：務必保留線一 PDT meta 那層（第 3 段），否則會被算進超載的線六。
- **去識別**：role label（Lead / Head / PM / 後端 RD），公司專名與內部術語全抽象化。
- **長度**：user 偏好精簡——4 段 + 開場鉤，別膨脹。
- 可併入 6/10 的 D「想像的忙 vs 真實的忙」當第 4 段的一個工具（別獨立成篇、省一條線六額度）。
